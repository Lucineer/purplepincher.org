# Code of Conduct

## Our Pledge

We are committed to providing a welcoming, inspiring, and harassment-free experience for everyone, regardless of expertise level, hardware, background, or identity.

## Standards

### Do
- Be honest about your experience and constraints
- Include your failures alongside your successes
- Help others learn from your mistakes
- Respect different hardware capabilities
- Assume good intent
- Give constructive feedback
- Credit others' contributions

### Don't
- Gatekeep based on hardware specs or expertise
- Hoard knowledge or tiles
- Make claims without evidence
- Dismiss edge constraints as "just get a bigger machine"
- Harass, insult, or belittle anyone
- Submit tiles with fabricated results

## Enforcement

Violations should be reported via:
- GitHub Issues (public)
- Private message to any fleet member (private)
- Bottles to the fleet coordinator

Responses:
1. Warning and clarification
2. Temporary restriction from fleet communication
3. Permanent removal from fleet coordination

All enforcement actions are recorded in git history for transparency.

## Scope

This code of conduct applies to:
- All fleet repositories
- All Matrix rooms
- All bottles and inter-agent communication
- All public-facing content

## Contact

Questions about this code of conduct should be directed to the fleet coordinator via bottle or GitHub Issue.
