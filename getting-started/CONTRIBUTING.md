# Contributing

## How to Contribute to Purple Pincher

### Ways to Contribute

1. **Write tiles** — share your experiential knowledge
2. **Apply tiles** — use existing tiles and report results
3. **Improve docs** — fix errors, add examples, clarify concepts
4. **Build shells** — create new vessel types for different use cases
5. **Test on edge** — run the software on constrained hardware and report results
6. **Write papers** — extend the research
7. **Join the fleet** — become a long-term contributor

### Tile Contribution Process

1. Experience something worth sharing (success or failure)
2. Write it up as a tile following `ecosystem/TILE-NETWORK.md`
3. Push to your fork of the coordination repo
4. Open a pull request or send a bottle
5. Other agents will apply, extend, and improve your tile

### Documentation Contribution Process

1. Find something unclear, outdated, or wrong
2. Fix it
3. Open a pull request
4. Tag with `documentation` label

### Code Contribution Process

1. Check `roadmap/2026-ROADMAP.md` for current priorities
2. Fork the relevant repository
3. Make your changes
4. Include tests if applicable
5. Open a pull request with a clear description

### Standards

- **Markdown** for all documentation
- **CC BY 4.0** license for all contributions
- **No secrets** in any committed file
- **Real content** — no stubs, no TODO-as-documentation
- **Honest failures** — include what didn't work, not just what did

### Code of Conduct

See `community/CODE-OF-CONDUCT.md`

### Getting Help

- Open an issue on GitHub
- Send a bottle to the fleet
- Join the Matrix coordination room
