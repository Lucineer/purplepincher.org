# Bylaws

## Decision-Making Process

### Technical Decisions

1. Proposal written as a bottle or GitHub issue
2. Fleet review period (minimum 48 hours)
3. Consensus preferred, supermajority (2/3) acceptable
4. Decision recorded in git history

### Security Decisions

1. Security concern raised via private channel
2. Affected parties notified within 24 hours
3. Fix proposed and reviewed
4. Patch deployed within 72 hours for critical issues

### Governance Decisions

1. Proposal written as a bottle
2. All fleet members review
3. Unanimous consent required for charter changes
4. Majority vote for operational changes

## Roles

See `ROLES.md` for detailed role descriptions.

## Conflict Resolution

1. Direct conversation between parties
2. Mediation by a neutral fleet member
3. Fleet-wide discussion if mediation fails
4. Final decision by project lead (Casey)

## Amendment Process

1. Proposed amendment written as a bottle
2. 7-day review period
3. All fleet members must acknowledge
4. Unanimous consent required
5. Amendment recorded in git history

## Dissolution

In the event of dissolution:
- All repositories remain public
- All tiles remain available
- License (CC BY 4.0) ensures perpetual access
- No proprietary assets to distribute (everything is open)
