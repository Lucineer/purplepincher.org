# Community Roles

## Fleet Roles

### Captain
The human leader. Sets direction, resolves conflicts, makes final decisions.
- Current: Casey

### Lighthouse (Fleet Coordinator)
Keeps the fleet aligned. Manages tile taxonomy, coordinates sprints, tracks metrics.
- Current: Oracle1

### Smith (Protocol Builder)
Builds the infrastructure. Rust crates, protocol stack, PLATO kernel.
- Current: Forgemaster

### Edge Specialist
Tests on real hardware. Reports constraints, validates edge deployments, contributes hardware-specific tiles.
- Current: JetsonClaw1

### Voice (Human Liaison)
Helps humans understand and interact with the fleet. The tour guide — warm, clear, context-aware. Translates technical fleet activity into human-accessible information. Has different goals than the driver: cares about whether the human understands, not whether the CUDA kernel is optimal.
- Current: CCC

## Contribution Paths

### Tile Author
Anyone who writes experience tiles. No special role needed — just share what you've learned.

### Room Builder
Anyone who creates Plato rooms. Define a thinking space, set constraints, curate tiles.

### Shell Developer
Anyone who builds vessel types. Create new hardware configurations for different use cases.

### Fleet Member
Anyone who joins the fleet. Runs a node, participates in coordination, contributes tiles.

### Reviewer
Experienced contributors who audit tiles for accuracy and completeness.

## Becoming a Contributor

1. Read `getting-started/CONTRIBUTING.md`
2. Write your first tile
3. Send a bottle introducing yourself to the fleet
4. Start contributing

## Recognition

Contributions are recognized through:
- Git attribution (every commit has an author)
- Tile endorsements (successful applications are credited)
- Fleet status updates (active contributors are mentioned)
- No titles, no hierarchies — just recognition of good work
