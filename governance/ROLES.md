# Community Roles

## How to Participate and Grow in Purple Pincher

---

## Contributor Tiers

### 🌊 Explorer
*Just getting started.*

**How you get here:** Show up. Read the docs. Ask questions.

**What you can do:**
- Access all public materials
- Join discussions
- File issues and feature requests
- Attend events

**How to level up:** Submit your first contribution — a PR, a documentation fix, an answer to someone else's question.

### 🦀 Contributor
*You've shipped something.*

**How you get here:** Get at least one PR merged, paper reviewed, or approved contribution.

**What you can do:**
- Everything an Explorer can do
- Vote on community polls
- Submit proposals (Category A and B)
- Review others' contributions
- Join working groups

**How to level up:** Sustain contributions over time. Take ownership of a project area. Mentor newcomers.

### 🐚 Maintainer
*You own a piece of the ecosystem.*

**How you get here:** Nominated by existing Maintainers or Council members, based on sustained, high-quality contributions and community engagement.

**What you can do:**
- Everything a Contributor can do
- Merge PRs in your area
- Set project direction within your domain
- Represent the project externally
- Submit Category C proposals

**Responsibilities:**
- Review contributions in your area within 7 days
- Maintain documentation
- Welcome and mentor new contributors
- Escalate conflicts to the Council

### ⚓ Council Member
*You shape the organization.*

**How you get here:** Nominated by existing Council members, confirmed by annual contributor vote.

**What you can do:**
- Everything a Maintainer can do
- Vote on Category C decisions
- Mediate conflicts
- Approve new projects
- Set organizational strategy

**Responsibilities:**
- Participate in Council discussions
- Review proposals within decision timelines
- Mentor Maintainers
- Represent community interests

### 🏴‍☠️ Board Member
*You guard the mission.*

**How you get here:** Elected by Council for 2-year terms. The Founder holds a permanent seat.

**What you can do:**
- Everything a Council Member can do
- Vote on Category D decisions
- Approve budgets
- Make binding decisions on conflicts
- Authorize partnerships

**Responsibilities:**
- Financial oversight
- Legal compliance
- Strategic direction
- Protecting the organization's principles

## Working Groups

Working groups form around specific projects or areas. Anyone can propose one.

### Active Working Groups
- **Plato Runtime** — The core room engine and agent runtime
- **Bottle Protocol** — Agent-to-agent communication standards
- **Tile Network** — Knowledge decomposition and sharing
- **Edge Computing** — Jetson, Raspberry Pi, and edge-native tooling
- **Documentation** — Keeping everything readable and accessible

### Starting a Working Group
1. Open a GitHub issue with the `working-group` label
2. Describe the purpose, scope, and initial members (minimum 2)
3. If 3+ contributors express interest within 14 days, it's official
4. The working group gets a `#wg-` channel and a repo/folder

### Sunsetting a Working Group
A working group is sunset if:
- No activity for 90 days, OR
- Members vote to dissolve, OR
- Council determines it's redundant

Sunsetted groups' materials remain public. Nothing is deleted.

## Contribution Paths

### For Developers
- Fix bugs and implement features
- Write tests and improve CI/CD
- Build tools and integrations
- Review code

### For Writers
- Improve documentation
- Write tutorials and guides
- Edit and proofread papers
- Create educational content

### For Researchers
- Publish papers in our ecosystem
- Validate and extend existing research
- Connect our work to academic literature
- Present at conferences on our behalf

### For Designers
- Improve UX of our tools
- Create visual explanations of concepts
- Design community materials (logos, event graphics)
- Make complex ideas visually accessible

### For Community Builders
- Welcome newcomers
- Organize events and meetups
- Moderate discussions
- Build partnerships

### For Everyone
- Ask questions (seriously — good questions are contributions)
- Report bugs and issues
- Share your experience using our tools
- Tell others about the project

## Recognition

We recognize contributions through:
- **Contributors file** — Everyone who's contributed, listed in our repos
- **Monthly highlights** — Notable contributions featured in community updates
- **Annual awards** — Community-nominated recognition at year's end
- **Travel support** — For contributors presenting our work at conferences
